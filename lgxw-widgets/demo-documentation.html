Download and Install

For a no-fuss install follow these directions:
  1. Copy entire logixware-framework directory to webroot of target server/domain.
  2. Point your browser to /lgxw-pages/demo.html and verify that the page renders without browser console errors.
  3. Copy /lgxw-pages/demo.html to your "pages directory" (e.g., /pages).
  4. Point your browser to the copied demo.html (e.g., /pages/demo.html) and verify that the page renders without browser console errors.
  5. Start "building out your content" by copying (and modifying) /pages/demo.html and copying (and modifying) /lgxw-data files.

Logixware Dashboard Widgets and Features

Each widget of three architecture values:
  1. DEPENDENT
    2. The first is a widget that uses resources loaded by the enclosing page.
  2. INDEPENDENT
    3. The second is a widget that loads its own resources.
  3. EXTERNAL
    4. The third is a (pre-existing) site or page that, by definition, loads its own resources.

Because of the implications of loading resources, the second and third are rendered in an iframe. To convey this to the Logixware Dashboard Plugin, we employ a variation on the "standard" JSON format required by the plugin. In particular, JSON attributes that are "non-native to the plugin", must be placed in the JSON record's "metadata".

Create "custom widgets" in /widgets, either from scratch or by copying and modifying the demo widgets.

Each widget "inherits" the following functionality:
  1. Widget Header
    2. Represents standard functionality including a title, controls and widget menu.
  2. Widget Body
    3. Represents premary data/functionality unique to this widget.
  3. (Optional) Widget Footer
    4. Represents secondary functionality unique to this widget.
